## 1. 技术栈选型
基于“单机使用、数据隐私、本地持久化”的核心需求，采用以下技术栈以兼顾开发效率与用户体验：

### 1.1 核心架构
- **跨平台方案**: **Web + 本地服务器** FastAPI + SQLite 环境。 
- **前端框架**: Vue3 + JavaScript。
- **样式方案**: Tailwind CSS (快速构建原型效果)。
- **状态管理**: Pinia
- **图标/组件库**: Lucide Vue + Headless UI (保持轻量且可定制)。

### 1.2 存储选型
- **数据库**: **SQLite 3**。
  - 理由：关系型数据库便于处理“父子嵌套”和“多级评论”；单文件存储便于用户备份和迁移。
- **ORM**: SQLAlchemy。

## 2. 编码规范要求

### 2.1 命名规范
- **文件/目录**: 采用 `kebab-case` (如 `work-item-list.tsx`)。
- **组件/类**: 采用 `PascalCase` (如 `DailyLogSelector`)。
- **变量/函数**: 采用 `camelCase` (如 `updateTaskStatus`)。
- **常量**: 采用 `SNAKE_UPPER_CASE` (如 `MAX_TITLE_LENGTH`)。

### 2.2 注释与文档
- **JSDoc/TypeDoc**: 核心业务函数必须包含输入输出说明及副作用描述。
- **代码自解释**: 优先通过语义化命名减少注释数量。

### 2.3 核心原则
- **单一职责**: 每个组件或类仅负责一个明确的功能（如：`CommentList` 只负责展示评论，不负责处理状态变更逻辑）。
- **防御性编程**: 对用户输入的 200/5000 字符长度进行前端严格校验。
